Foundation · Section 01
What is an Operating System?
The OS is the foundation of everything. Commands run through it.
Definition

An Operating System is software that manages hardware and executes programs. Every command you run goes through it.

What the OS does
⚙️ Manages CPU, memory, disk
📁 Controls file system access
▶️ Executes programs & commands
🔐 Enforces user permissions
🐧
Linux
Most servers run Linux
🪟
Windows
Common on enterprise
🍎
macOS
Unix-based like Linux
💡 Why this matters: Web servers run on Linux. OS Command Injection attacks target Linux commands directly on the server.
Foundation · Section 02
What is an OS Command?
Commands are instructions given directly to the OS. They have high privileges.
Common Linux Commands
ls       # list files in directory
pwd     # show current directory
whoami  # show current user
cat     # print file contents
ping    # test network connection
id      # show user info
Interactive Terminal — Try a command
terminal
root@server:~$ # select a command and click Run
⚠️ OS commands have direct access to the system — files, network, users, processes. An attacker running OS commands can do anything the server user can do.

Input & Vulnerabilities · Section 06
Safe vs Unsafe Code
The difference between a secure app and a vulnerable one is how input is handled.
✓ SAFE — Sanitized Input
// PHP Example $ip = escapeshellarg ($_GET['ip']);
system("ping -c 1 " . $ip);

// Python Safe subprocess.run (["ping","-c","1", ip])
✗ UNSAFE — Direct Concatenation
// PHP — VULNERABLE $ip = $_GET['ip']; // NO sanitization!
system("ping -c 1 " . $ip);

// Python — VULNERABLE os.system ("ping -c 1 " + ip)
Safe code does:
Wraps input in quotes, escapes special characters, treats input as data — NOT as shell code
Unsafe code does:
Concatenates raw input directly into a shell command — attackers can inject anything
Input & Vulnerabilities · Section 07
What is OS Command Injection?
The moment user input escapes its intended role and becomes OS instructions.
Definition
"User input modifies OS command execution, allowing arbitrary commands to run on the server."
Normal Case
Input: 8.8.8.8
↓
Command: ping -c 1 8.8.8.8
↓
Output: ping result
Injected Case
Input: 8.8.8.8; whoami
↓
Command: ping -c 1 8.8.8.8; whoami
↓
Output: ping + username
Input & Vulnerabilities · Section 08
Why Does This Work?
The OS cannot tell if special characters came from the developer or an attacker.
The Core Problem
01
OS commands accept special characters like ; | && $( ) as instruction separators
02
Applications concatenate user input directly into shell commands without filtering
03
The shell interprets the separator and executes what follows as a new command
What the shell sees
ping -c 1 8.8.8.8 ; whoami
↑ original command
↑ separator
↑ injected command

Exploitation · Section 10
Attacker Mindset
Three questions every attacker asks before attempting injection.
🎯 Question 1: Can I control the input?
Look for URL parameters (?ip=, ?host=), form fields, and any user-controlled data sent to the server. If you can change the value — you have a potential vector.
🔍 Question 2: Is that input passed to the OS?
Look for features like ping tools, DNS lookups, file operations, image processing, or any functionality that might call a system command. Burp Suite intercept helps you see how the app processes input.
💉 Question 3: Can I append additional commands?
Test with a separator followed by a recognizable command: ; whoami or ; sleep 5. If output changes or there's a delay — injection confirmed.

Defense · Section 16
Defense & Mitigation
Prevention is simple — but developers must know the rules.
Attack Simulation
server response
Toggle protection and click Launch Attack...
Mitigation Techniques
01
Avoid Shell Execution
Use language APIs that don't invoke a shell. Pass arguments as arrays, not strings.
subprocess.run(["ping", "-c", "1", ip])
02
Validate & Whitelist Input
Accept only known-good patterns. For an IP field, only allow digits and dots.
if not re.match(r'^\d+\.\d+\.\d+\.\d+$', ip): reject()
03
Reject Special Characters
Block or escape characters: ; | & $ ( ) ` > < !
04
Least Privilege
Run web apps as non-root with minimal OS permissions. Even if injected, damage is limited.
